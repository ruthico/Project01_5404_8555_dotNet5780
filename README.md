## 📝 אודות הפרויקט
פרויקט זה פותח במסגרת מעבדת .NET ושפת C# (שנת הלימודים תש"ף - 5780). המערכת נבנתה בארכיטקטורת שכבות (N-Tier Architecture) ומממשת עקרונות תכנות מונחה עצמים (OOP) מתקדמים, עיצוב תוכנה מבוסס תבניות (Design Patterns), ועבודה מול ממשק משתמש גרפי עשיר.

המערכת מנהלת את תהליכי הלוגיקה, המעקב והרישום של הפרויקט בהתאם לדרישות המדויקות של קורס המעבדה.

## 🏛️ ארכיטקטורת המערכת ומבנה הפתרון (Solution Layout)
הפתרון מחולק ל-4 שכבות לוגיות נפרדות (Projects) המפרידות בצורה מוחלטת בין תצוגה, לוגיקה עסקית וגישה לנתונים:

1. **`PL (Presentation Layer) - UI`**
   - ממשק משתמש גרפי מודרני המבוסס על **WPF (Windows Presentation Foundation)** וקובצי XAML.
   - מימוש עקרונות חווית משתמש, שימוש ב-Data Binding, מנועי עיצוב (Styles & Templates), פקדים מותאמים אישית, וחלונות אינטראקטיביים (CRUD).
   
2. **`BL (Business Logic Layer)`**
   - שכבת הלוגיקה העסקית של המערכת.
   - אחראית על וולידציות, חישובים מורכבים, חוקים עסקיים של המערכת, והטלת חריגות (Exceptions) מותאמות אישית במקרה של הפרת תנאים.
   - מתווכת בין שכבת התצוגה לשכבת הנתונים באמצעות ממשק (`IBL`).

3. **`DAL (Data Access Layer)`**
   - שכבת הגישה לנתונים האחראית על שמירה, שליפה ועדכון של המידע.
   - **מנגנון שמירה כפול:** תמיכה בשמירה בזיכרון המקומי (`DS_XML` / `Lists`) לצד עבודה מול קובצי **XML** מובנים הכוללים יצירה אוטומטית של קבצים בהרצה ראשונית.
   - מימוש תבנית העיצוב **Repository Pattern**.

4. **`BE (Business Entities)`**
   - שכבת הישויות המכילה את מחלקות הבסיס (Data Contracts / POCO Classes) המשותפות לכל השכבות בפתרון.
   - המחלקות כוללות תכונות (Properties), בנאים (Constructors) ומימוש פונקציות בסיסיות (כגון `ToString`).

## 🛠️ טכנולוגיות ויכולות מיושמות
- **שפה:** C# (גרסה תואמת .NET Framework 4.7.2 / .NET Core בהתאם לדרישות שנת תש"ף).
- **ממשק משתמש:** WPF, XAML, Windows Forms Dialogs.
- **ניהול נתונים:** קובצי XML, Serialization/Deserialization, LINQ to XML.
- **שאילתות מתקדמות:** שימוש נרחב ב-**LINQ** (שפת שאילתות מובנית) לסינון, קיבוץ (Grouping) ומיון נתונים בשכבת הלוגיקה.
- **תבניות עיצוב (Design Patterns):**
  - **Singleton:** להבטחת מופע יחיד של מחלקות ה-BL וה-DAL לאורך כל מחזור חיי האפליקציה.
  - **Factory Method:** ליצירה דינמית של מופעי ה-DAL (בחירה בין רכיב מקומי לרכיב XML).
